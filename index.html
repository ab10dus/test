From Friction to Foresight

A strategic proposal to evolve the VisNet NIFT platform and empower DNOs for the new era of energy connections.

The Core Insight: From Technical Tool to Strategic Partner

The feedback that the workflow is "too time-consuming" and lacks "the right value" is not a simple feature request. It is a signal that the very definition of "value" has changed for our DNO customers. Under the pressure of the Connections Action Plan (CAP), they no longer need just a technically accurate assessment tool; they need a strategic partner that helps them make faster, smarter, and more cost-effective decisions to manage their connection queues.

The Old Definition of Value

● Focus: Technical Accuracy

● Output: A single, static "pass/fail" result

● Process: Manual, linear, and slow

● User Goal: "Is this connection technically feasible?"

The New Definition of Value (Post-CAP)

● Focus: Strategic Decision Support

● Output: A comparison of costed, viable scenarios

● Process: Automated, iterative, and rapid

● User Goal: "What is the fastest, most cost-effective way to connect this LCT while managing my queue?"

Phase 1: Deconstructing the Problem & Validating Assumptions

(Sprints 1-2)

Testable Hypotheses

We will translate the vague feedback into specific, testable hypotheses. This rigor ensures we are solving the right problems and building a solution that delivers true value in the new regulatory environment.

H1: Data Friction

Engineers spend >50% of their time on manual data aggregation from disparate systems (GIS, spreadsheets) before an assessment can even begin.

H2: Lack of Strategic Context

The current output (e.g., voltage drop) is insufficient. Users need indicative reinforcement costs and CAP-compliant options to make effective decisions.

H3: Inability to Model Scenarios

The tool cannot efficiently model or compare alternative solutions (e.g., firm vs. non-firm connections), which is now essential for queue management.

From Epic to User Story: Defining the MVP

To ensure clarity and focus, we will define a tightly-scoped MVP: the "NIFT Connections Accelerator". This section breaks down the primary epic into its core components and provides example user stories.

EPIC: Build the Connections Accelerator MVP

Create a streamlined, guided workflow within the NIFT platform that dramatically reduces the time and effort required to assess a standard domestic LCT connection.

Component 1: Automated Data Ingestion

  • User Story: "As a Connection Engineer, I want to input a location (postcode or coordinates), so that the tool automatically pulls the relevant network topology and asset data from our GIS via an API, eliminating manual entry."

Component 2: Guided Assessment UI

  • User Story: "As a Connection Engineer, I want a single-page, step-by-step workflow to input connection details (e.g., EV charger kW), so that I can complete an assessment without navigating multiple screens."

Component 3: Contextual Results Display

  • User Story: "As a Connection Engineer, when an assessment identifies a constraint, I want to see an indicative reinforcement cost alongside the technical data, so I can immediately understand the commercial implications."

Component 4: Pilot Feedback Mechanism

  • User Story: "As a pilot user, I want a simple way to provide feedback (e.g., a one-click satisfaction rating and comment box) after each assessment, so that I can help the product team improve the tool."
